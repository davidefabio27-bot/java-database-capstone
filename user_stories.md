# User Stories

This document contains user stories for the three main user types of the Smart Clinic system: Admins, Patients, and Doctors.  
Each story follows the standard format:  
**As a [role], I want [goal], so that [benefit].**

1. **As an Admin, I want to register new doctors**, so that they can start managing their availability and appointments.
2. **As an Admin, I want to manage system users (enable/disable accounts)**, so that I can control who has access to the system.
3. **As an Admin, I want to view and update doctor profiles**, so that I can keep doctor information accurate.
4. **As an Admin, I want to view the list of all patients**, so that I can help doctors or patients when needed.
5. **As an Admin, I want to view all appointments in the system**, so that I can monitor overall scheduling activity.
6. **As an Admin, I want to reset or update system configurations**, so that the system operates correctly.

1. **As a Patient, I want to create an account**, so that I can access the clinic‚Äôs appointment services.
2. **As a Patient, I want to log in securely**, so that I can manage my medical information.
3. **As a Patient, I want to view available doctors and their schedules**, so that I can select an appropriate appointment time.
4. **As a Patient, I want to book an appointment**, so that I can visit a doctor at my preferred date and time.
5. **As a Patient, I want to view my upcoming and past appointments**, so that I can track my medical history.
6. **As a Patient, I want to cancel or reschedule an appointment**, so that I can manage unexpected changes.
7. **As a Patient, I want to view my prescriptions**, so that I can follow my doctor's treatment instructions.

1. **As a Doctor, I want to log in securely**, so that I can access my appointments and patient information.
2. **As a Doctor, I want to set or update my availability**, so that patients can book appointments only when I am free.
3. **As a Doctor, I want to view my scheduled appointments**, so that I can prepare for upcoming consultations.
4. **As a Doctor, I want to view patient medical details**, so that I have the necessary information for each visit.
5. **As a Doctor, I want to add or update prescriptions**, so that patients receive accurate treatment information.
6. **As a Doctor, I want to mark appointments as completed**, so that the system keeps accurate medical and scheduling records.
